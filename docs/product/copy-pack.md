## 2. GitHub Repo Intro (kurz)

ARIA is a lean, modular, self-hosted AI assistant built for people who want control instead of platform sprawl.

It combines a browser-first chat UI, structured memory, configurable skills, and secure connections to real systems. ARIA is intentionally not trying to be a giant OpenWebUI-style suite. It is designed to stay understandable, extensible, and practical to run.

---

## 3. Repo Intro (mittel)

ARIA is a browser-first AI assistant for people who want a capable local system without the weight of a large AI platform.

It combines:

- chat
- structured memory
- configurable skills
- modular connections
- explicit security and role boundaries

ARIA is built to stay small, inspectable, and useful. Instead of hiding behaviour behind a giant framework stack, it keeps the core understandable and moves toward capability-based routing so that new features and later user-defined extensions remain realistic.

## 4. Product Overview (kurzer Pitch)

ARIA is a compact local AI assistant that can remember context, talk to real systems, and stay operable by one person or a small team.

It is designed for homelabs, self-hosted AI setups, internal assistants, and small automation environments where control, clarity, and modularity matter more than feature bloat.
