Jeden Schritt von oben nach unten abarbeiten. Blau unterstrichene Links oeffnen und danach hierher zurueckkehren.

1
Google Calendar API aktivieren
  1. Google Calendar API ↗ oeffnen
  2. Schaltflaeche Aktivieren klicken
  3. Seite laedt neu - oben erscheint «API aktiviert» ✓
2
OAuth-Branding einrichten
  1. OAuth Branding ↗ oeffnen
  2. Schaltflaeche Jetzt starten klicken
  3. Feld App-Name: beliebigen Namen eintragen, z.B. «ARIA»
  4. Feld Nutzer-Support-E-Mail: deine E-Mail-Adresse auswaehlen
  5. Ganz unten Kontakt-E-Mail des Entwicklers: gleiche E-Mail-Adresse eintragen
  6. Weiter -> Weiter -> Erstellen klicken
3
Dich selbst als Testnutzer eintragen
  1. Testnutzer (Audience) ↗ oeffnen
  2. Schaltflaeche Nutzer hinzufuegen klicken
  3. Deine eigene Google-E-Mail-Adresse eintragen
  4. Hinzufuegen -> Speichern klicken
4
OAuth-Client anlegen
  1. OAuth Clients ↗ oeffnen
  2. Schaltflaeche Client erstellen klicken
  3. Anwendungstyp: Webanwendung auswaehlen
  4. Name: beliebig, z.B. «ARIA Web Client»
  5. Abschnitt Autorisierte Weiterleitungs-URIs -> URI hinzufuegen klicken
  6. Genau diesen Wert einfuegen: https://developers.google.com/oauthplayground
  7. Erstellen klicken
  8. Popup erscheint - Client-ID und Client-Secret notieren oder JSON herunterladen
5
Refresh-Token holen
  1. OAuth Playground ↗ oeffnen
  2. Oben rechts Zahnrad-Symbol klicken
  3. Haekchen bei Use your own OAuth credentials setzen
  4. OAuth Client ID und OAuth Client Secret aus Schritt 4 eintragen - Zahnrad schliessen
  5. Links im Textfeld Input your own scopes diesen Wert eintragen: https://www.googleapis.com/auth/calendar.readonly
  6. Schaltflaeche Authorize APIs klicken
  7. Dein Google-Konto auswaehlen
  8. Warnung «App nicht verifiziert» erscheint -> Erweitert klicken -> Weiter zu ARIA klicken -> Zulassen klicken
  9. Zurueck im Playground: Schaltflaeche Exchange authorization code for tokens klicken
  10. Links unter Refresh token den langen Code kopieren und aufbewahren
Tritt die Warnung «App nicht verifiziert» nicht auf, sondern eine Fehlermeldung, pruefe ob deine E-Mail in Schritt 3 korrekt eingetragen ist.
6
Werte in ARIA eintragen
  1. Client-ID aus Schritt 4 einfuegen
  2. Client-Secret aus Schritt 4 einfuegen
  3. Refresh-Token aus Schritt 5 einfuegen
  4. Kalender-ID: fuer den Hauptkalender einfach primary eingeben
  5. Fuer einen anderen Kalender: Google Calendar Einstellungen ↗ oeffnen -> Kalender anklicken -> Einstellungen und Freigabe -> ganz unten «Kalender integrieren» -> Kalender-ID kopieren
  6. Profil speichern
Nach dem Speichern testet ARIA die Verbindung sofort. Bei Fehler: pruefe ob Schritt 3 (Testnutzer) korrekt ausgefuellt ist.
